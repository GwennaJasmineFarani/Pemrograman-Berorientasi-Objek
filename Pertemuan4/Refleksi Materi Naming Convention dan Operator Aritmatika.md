**Refleksi Materi Naming Convention dan Operator Aritmatika**

Pada materi ini saya dapat memahami bahwa **Naming Convention** adalah
aturan penamaan dalam Java agar kode rapi, konsisten, dan mudah
dipahami. Class ditulis dengan *PascalCase* seperti
P041NamingConvention, variabel dan method menggunakan *camelCase*
seperti studentAge, dan konstanta menggunakan huruf kapital seperti PI.
Penamaan yang jelas penting karena memudahkan membaca kode dan
mengurangi kesalahan.

Saya juga memahami penggunaan **Operator Aritmatika** yaitu +, -, \*, /,
dan % untuk melakukan perhitungan. Hasil operasi bergantung pada tipe
data. Jika menggunakan int, pembagian tidak menghasilkan desimal. Untuk
mendapatkan hasil desimal, perlu menggunakan double atau casting seperti
(double) a / b.

Dari Latihan tersebut, saya belajar menghitung operasi dasar dan
menampilkan hasilnya ke terminal. Kesulitan yang saya alami adalah
kesalahan penulisan tipe data seperti string dan inr, serta memahami
pembagian desimal. Namun setelah diperbaiki, program dapat berjalan
dengan benar.
